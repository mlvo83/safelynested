Onboarding a Charity Donor

A guide for charity staff setting up donors in SafelyNested
What this guide covers. How a Charity Partner or Charity Facilitator submits a request to set up a donor — either by linking an existing donor in the system to your charity, or by registering a brand new donor. All requests are reviewed by a SafelyNested administrator before the donor is added.

About donors on SafelyNested

A "donor" is an individual or organization who has given (or plans to give) money to your charity. Recording donors and their donations on the platform unlocks two things:

  • You can record their donations and pay the platform fee through the system.
  • Their donations can be used to fund actual bookings — nights of housing for families — with full traceability from donation through to a real stay.

Because donor records affect financial flows and donor privacy, charities don't create donors directly. You submit a request, and a SafelyNested administrator reviews and approves it.

Two kinds of requests:
  • Link Existing Donor — the donor already exists somewhere else in SafelyNested (perhaps they've also given to another charity). You ask the admin to link them to your charity too.
  • Register New Donor — the donor isn't in the system at all. You provide their details and the admin creates a brand-new donor record on approval.

The donor setup flow

1

Open the "Set Up a Donor" page

Two ways to get there from your charity dashboard:

  • From the Dashboard, click the green Set Up a Donor quick-action button.
  • Or, in the left sidebar click Donors, then click Set Up a Donor at the top of the list.

You'll land on a page with two sections stacked vertically:

  • Top: Search Existing Donors
  • Bottom: Register New Donor

Always start with the search — it prevents duplicate donor records.

PATH A — LINK EXISTING DONOR

Use this if you suspect the donor may already be in the system (for example, they've previously given to another SafelyNested charity, or another team member registered them).

2

Path A: search for the donor

In the search box at the top of the page, type at least 2 characters — name, email, or business name. Results appear automatically as you type.

Each result row shows:

  • Name — the donor's display name.
  • Email.
  • Type — Individual or Business / Organization.
  • Status — Verified or Pending.
  • Action — either a Select button, or an Already Linked badge if they're already part of your charity.

Click Select next to the right donor.

Tip: If you can't find the donor after a thorough search, scroll down and use Path B (Register New Donor) instead.
3

Path A: submit the link request

After selecting, a green confirmation card appears showing the donor's name and email. Below it:

  • Notes (optional) — a short explanation of why you want this donor linked. Helpful context for the admin (e.g., "The Smith family has supported us for 3 years — recently moved their giving through SafelyNested").

Click Submit Link Request. You'll see a green success message and be taken to your Donor Setup Requests page where the new request is listed with status Pending and a request number in the format DSR-YYYY-NNNN. Save this number.

PATH B — REGISTER NEW DONOR

Use this if your search returned no matching donor.

4

Path B: choose the donor type

Scroll to the Register New Donor section. At the top, choose:

  • Individual (the default) — for a person.
  • Business / Organization — for a company, foundation, faith group, or similar entity.

The form below adapts to the type you choose.

5

Path B: fill in the donor's details

Required fields are marked with a red asterisk on the form. Other fields are optional but help the admin verify the donor and help you stay in touch with them later.

For an individual

  • First Name *
  • Last Name *

For a business / organization

  • Company / Organization Name *
  • Contact Person Name (the person at the organization you deal with)
  • Tax ID / EIN

Contact information (both types)

  • Email
  • Phone

Mailing address

  • Street Address, City, State, ZIP Code

Preferences

  • Anonymity Preference — whether the donor wants to be acknowledged publicly, listed by name only with the charity, or stay fully anonymous. Pick what they've told you.
  • Preferred Contact Method — email, phone, mail, or no contact.

Final field

  • Notes — freeform notes for the SafelyNested admin reviewer (e.g. context for why the donor is being added, how the charity received the donation, etc.).
6

Path B: submit the new donor request

The Requesting Charity field at the bottom is filled in automatically and locked — it confirms which charity the donor will be linked to.

Click Submit New Donor Request. You'll see a green success message and be taken to your Donor Setup Requests page. The request number (format DSR-YYYY-NNNN) appears in the list with status Pending.

Duplicate prevention: The system will reject a new-donor request if there's already a pending request with the same email at your charity, or if the existing donor is already linked to your charity. If you hit this, check your existing requests — the one you need is probably already there.
7

Wait for admin review

A SafelyNested administrator will review your request. They have three actions available:

  • Approve a Link request — the existing donor is immediately linked to your charity.
  • Approve a New Donor request — a new donor record is created in the system, given an auto-generated username (e.g. donor_first.last_NNNN) and a temporary password, and immediately linked to your charity.
  • Reject the request — with a written reason. You'll be able to read the reason on the requests page.

Check the status of your requests anytime by clicking Donors in the sidebar, then View My Requests, or visiting /charity-partner/donors/setup-requests. Filter by status (Pending, Approved, Rejected) using the buttons at the top.

8

Once approved: the donor appears in your list

An approved donor (whether linked or newly created) shows up in your charity's donor list at /charity-partner/donors. From there you can:

  • See their contact information and preferences.
  • Record donations they've given to your charity.
  • Use those donations to fund bookings for families needing housing.
For newly created donors: If your donor needs to log in to SafelyNested themselves (for example, to view their donor dashboard and impact reports), the SafelyNested admin will provide their generated username and a temporary password through a separate channel. Coordinate with the admin to ensure this reaches the donor securely — never share login credentials over insecure email.

Privacy and donor information

Donor information lives in SafelyNested with these protections:

  • Anonymity preferences are honored: if a donor selects "Fully Anonymous", their name is suppressed in donor-facing reports.
  • Other charities cannot see your donors by default; only when an admin explicitly links them.
  • Family / participant information is never visible to donors — donors see only the locations, dates, and number of nights their donations funded, never the names or details of the families staying.

Common issues

ProblemWhat to do
Search returns "Already Linked" for a donor I'm trying to add.That donor is already in your charity's donor list. Go to Donors in the sidebar — you'll find them there.
I get an error: "A pending request already exists...".Open Donors → View My Requests and look for the pending request — it's probably one you submitted earlier. No need to submit again.
I submitted a request days ago and it's still pending.Contact your SafelyNested administrator with the request number (DSR-YYYY-NNNN). For requests that are blocking active fundraising or booking funding, mention the urgency.
My request was rejected and I disagree.The rejection reason is shown on the requests page. You can submit a new request addressing the concern, or contact the admin to discuss.

Help and support

Donor records can directly fund housing for families in need. If a request is blocked or delayed and is preventing you from funding a booking, contact your SafelyNested administrator and explain the situation — we'll prioritize accordingly.
